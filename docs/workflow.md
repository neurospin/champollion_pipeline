# Champollion Pipeline — Internal Workflow Reference

This document describes the internal mechanics of the two external submodules:
`external/cortical_tiles` (crop generation) and `external/champollion_V1` (self-supervised training and inference).

---

## 1. cortical_tiles — Sulcal Region Crop Generation

### Entry point

```
generate_sulcal_regions.py  -d <output_dir>  --path_to_graph …  --path_sk_with_hull …  --njobs N
```

Invoked by `src/run_cortical_tiles.py`, which first writes a resolved `pipeline_loop_2mm.json` into the output directory and then calls this script from its own directory.

### Outer loop

`generate_sulcal_regions()` iterates over the Cartesian product:

```
28 regions  ×  {L, R}  ×  {skeleton, foldlabel, extremities}
```

Each combination is handled by a `RegionPipelineRunner`, which calls `run_with_params()` from `generate_one_sulcal_region.py`.

Two regions get special treatment:
- `OCCIPITAL` and `F.C.L.p.-subsc.-F.C.L.a.-INSULA.` use `threshold=1` (less conservative mask dilation).
- `F.C.L.p.-subsc.-F.C.L.a.-INSULA.` on the left side also uses `threshold=1`.

### Per-region pipeline (generate_one_sulcal_region.py)

Each `run_with_params()` call executes the following steps in sequence. All intermediate files are per-subject NIfTIs written to subdirectories of `output_dir`.

#### Step 1 — Compute mask (`compute_mask`)

Builds a canonical **3-D bounding-box mask** for the region in ICBM2009c space at 2 mm isotropic resolution.

- Reads the `masks_version` field from `pipeline_loop_2mm.json` (e.g. `canonical_25`) to select the pre-computed population mask NIfTI.
- Dilates / thresholds the mask to define the crop bounding box.
- Output: `{output_dir}/crops/2mm/{region}/mask/{side}mask_cropped.nii.gz` + `.minf` sidecar (stores `sizeX/Y/Z`).

The mask is computed **once** and re-used for all subjects. It determines the crop dimensions that appear in the per-region YAML files generated by `create_dataset_config_files.py`.

#### Step 2 — Generate raw per-subject volumes

Depending on `input_type`, one of three generators runs:

| input_type | Function | Source | Output per subject |
|---|---|---|---|
| `skeleton` | `generate_skeletons` | Morphologist `.arg` graph | skeleton voxels in native space (`.nii.gz`) |
| `foldlabel` | `generate_foldlabels` | Morphologist `.arg` graph | labelled fold voxels (`.nii.gz`) |
| `extremities` | `generate_extremities` | skeleton-with-hull NIfTI | extremity voxels (`.nii.gz`) |

QC filtering happens here: subjects listed as failing in the TSV pointed to by `skel_qc_path` are skipped before processing.

#### Step 3 — Resample to ICBM2009c (`resample_files`)

Each per-subject NIfTI is resampled from native Morphologist space to **ICBM2009c at 2 mm** using the subject-specific affine transform generated by `generate_ICBM2009c_transforms`.

#### Step 4 — Mask foldlabels / extremities (foldlabel and extremities only)

- `mask_foldlabel_files` / `mask_extremities_files`: zero-fills voxels outside the canonical mask so that all subjects share the same active spatial support.

#### Step 5 — Crop (`generate_crops`)

Applies the mask bounding box computed in Step 1 to extract the region of interest from each resampled volume.
Output: `{output_dir}/crops/2mm/{region}/mask/{side}{input_type}_cropped.nii.gz` per subject.

#### Step 6 — (Optional) Distance-to-bottom (`generate_distbottom_crops`)

Generates a distance-transform map from the fold bottom. Skipped when `--skip-distbottom` is passed (unused during inference).

#### Step 7 — Aggregate to NumPy

After all subjects are cropped, a final pass collects every per-subject NIfTI into:

```
{output_dir}/crops/2mm/{region}/mask/{side}skeleton.npy          # (N, 1, X, Y, Z) float32
{output_dir}/crops/2mm/{region}/mask/{side}skeleton_subject.csv  # N rows, one subject ID per row
```

Champollion training and inference read exclusively from these `.npy` + `_subject.csv` pairs.

### Config file: pipeline_loop_2mm.json

Written by `run_cortical_tiles.py` into the output directory before the script runs.
Key fields resolved at runtime:

| Field | Meaning |
|---|---|
| `graphs_dir` | Root directory of Morphologist subject folders |
| `output_dir` | Where derivatives are written |
| `path_to_graph` | Sub-path under each subject to the `.arg` graph |
| `path_to_skeleton_with_hull` | Sub-path to the skeleton-with-hull NIfTI |
| `masks_version` | Which canonical mask set to use (`canonical_25` by default) |
| `skel_qc_path` | Path to QC TSV; empty string = no filtering |
| `skip_distbottom` | Boolean; set to `true` to skip Step 6 |

## 2. champollion_V1 — Self-Supervised Training and Embedding Inference

### Config system (Hydra)

The script `contrastive/train.py` is decorated with `@hydra.main(config_name='config', config_path='configs')`.
Hydra composes the final config from several config groups:

```
configs/
  config.yaml                          # root: references all groups
  platform/cuda_not_brainvisa.yaml     # device = 'cuda', no BrainVISA env
  mode/encoder.yaml                    # training mode: self-supervised encoder
  dataset/<DATASET_NAME>/<region>.yaml # per-region paths (generated by champollion_pipeline)
  dataset_localization/local.yaml      # dataset_folder: absolute root on this machine
```

`dataset_folder` is a Hydra variable interpolated into every path in the region YAML:

```yaml
# local.yaml
dataset_folder: /neurospin/dico/data/human

# SC-sylv_left.yaml (generated)
numpy_all: ${dataset_folder}/UKBioBank/derivatives/deep_folding-2025/crops/2mm/S.C.-sylv./mask/Lskeleton.npy
```

Region YAMLs are generated by `src/generate_champollion_config.py` (via `create_dataset_config_files.py`). The `--external_crops` flag allows pointing at a dataset with a non-standard derivatives path.

### Training (`train.py` / `src/train_champollion.py`)

#### Data loading — DataModule_Learning

1. Reads `{numpy_all}` (`.npy`, shape `[N, 1, X, Y, Z]`) and the companion `_subject.csv` into memory.
2. Splits subjects into train / validation sets.
3. Each `__getitem__` call applies **two independent augmentation chains** to the same crop, producing a positive pair `(x_i, x_j)` for contrastive learning.

#### Augmentations (`augmentations.py`)

Spatial and structural augmentations applied stochastically per sample:

| Augmentation | Description |
|---|---|
| Random trim | Removes fold branches above a depth threshold (`trimdepth`) |
| Random cutout | Zeros a random 3-D patch (occlusion) |
| Gaussian noise | Adds voxel-level noise (`sigma_noise`) |
| Random flip | Left–right reflection |
| Random translation | Sub-voxel 3-D shift |
| Random rotation | Small-angle 3-D rotation |
| Keep extremity | Optionally forces extremity voxels to survive trimming |

#### Model — ContrastiveLearnerFusion

```
input (1, X, Y, Z)
    └─ backbone (encoder)
         ├─ DenseNet   (default)
         ├─ ConvNet
         ├─ ResNet
         └─ ConvNeXt
    └─ representation vector  [backbone_output_size dims]   ← saved as embedding
    └─ projection head (MLP)
    └─ output vector  [num_outputs dims]                    ← used for loss only
```

Multiple regions can share a **single encoder** (`multiregion_single_encoder=True`) or use **separate encoders** fused at the projection head.

#### Loss

Two loss functions are supported (configured via `mode`):

| Loss | Description |
|---|---|
| **NT-Xent** (SimCLR) | Normalised temperature-scaled cross-entropy over positive pairs in a batch |
| **Barlow Twins** | Minimises redundancy between dimensions of the projection; no negative pairs needed |

#### Training loop (PyTorch Lightning)

- `pl.Trainer` with TensorBoard logging (`logs/`) and optional W&B.
- Early stopping on `val_loss` (patience = `early_stopping_patience`).
- Checkpoint saved to `hydra.run.dir/best_model.pt`.
- Hydra writes run config to `hydra.run.dir/.hydra/config.yaml`.

### Inference (`generate_embeddings.py` / `src/generate_embeddings.py`)

1. Loads the trained model from `best_model.pt` (or a HuggingFace repo / local archive).
2. Instantiates `DataModule_Evaluation` — same `.npy` loading, no augmentation, full dataset.
3. Runs a forward pass through the **backbone only** (projection head is discarded).
4. Saves one `full_embeddings.csv` per region/hemisphere:
   - Rows: subjects (identified by the `_subject.csv` IDs)
   - Columns: `dim1 … dimD` where `D = backbone_output_size`

The pipeline runs **56 inference passes** (28 regions × 2 hemispheres), one per trained model fold.
`put_together_embeddings.py` then collects all 56 CSVs into a single output directory.

## Data flow summary

```
T1 MRI
  │
  ▼ Morphologist (generate_morphologist_graphs.py)
sulcal graphs (.arg)
  │
  ▼ cortical_tiles (run_cortical_tiles.py)
  │   for each of 28 regions × {L,R} × {skeleton, foldlabel, extremities}:
  │     compute_mask → generate raw volumes → resample → crop
  │     → aggregate → {region}/mask/{side}skeleton.npy + _subject.csv
  │
  ▼ generate_champollion_config.py
  │   region YAMLs + local.yaml (dataset_folder)
  │
  ▼ champollion_V1 training (train_champollion.py)  ×56 region/side models
  │   DataModule → augmented positive pairs
  │   ContrastiveLearnerFusion (backbone + projection head)
  │   NT-Xent / Barlow Twins loss → best_model.pt
  │
  ▼ champollion_V1 inference (generate_embeddings.py)  ×56
  │   backbone forward pass (no projection head)
  │   → full_embeddings.csv per region/side
  │
  ▼ put_together_embeddings.py
combined embeddings directory
  │
  ▼ generate_snapshots.py
UMAP projection onto 42k-subject UKBioBank reference cloud
```
